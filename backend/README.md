## 🔥 확장성을 고려한 빌딩

> 제품의 지속 가능성을 위해 배우고, 관심을 가져야 함
> 
- 🔥 마이그레이션 전략
    - `단계적 기능 축소` (Graceful Degradation)
    - `스로틀링` (Throttling)
    - `Backpressure`
    - `서킷 브레이커` (Circuit Breaker)
- 차이점을 이해하기
    - `Instrumentation` vs `Monitoring` vs `Telemetry`
- 🔥 수직적 확장 vs 수평적 확장
- 🔥 관찰 가능성을 고려한 확장
    
    > log를 남기고, 상태를 관측 가능한 아이템을 사용하는 방법 등을 통해문제 발생 시 디버깅을 돕고 이슈를 해결하도록 함
    >
